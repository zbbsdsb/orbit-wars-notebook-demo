# Proximal Policy Optimization Algorithms

**Authors**: John Schulman, Filip Wolski, Prafulla Dhariwal, Alec Radford, Oleg Klimov
**Institution**: OpenAI
**Year**: 2017
**arXiv**: [1707.06347](https://arxiv.org/abs/1707.06347)

## Core Contribution
A family of policy optimization methods that are simple to implement, have good sample efficiency, and stable training dynamics. PPO strikes a balance between ease of implementation, sample efficiency, and reliable performance, becoming the default RL algorithm for many applications.

## Key Techniques
- **Clipped Surrogate Objective**: Limits the policy update ratio to [1-ε, 1+ε] (typically ε=0.2), preventing destructively large updates
- **Multiple Epochs**: Reuses each batch of data for multiple SGD epochs (typically 3-10), improving sample efficiency
- **GAE (Generalized Advantage Estimation)**: Reduces variance of advantage estimates using λ (typically 0.95)
- **Actor-Critic Architecture**: Jointly trains a policy network (actor) and a value network (critic)
- **Entropy Bonus**: Adds entropy regularization to encourage exploration
- **Vectorized Environments**: Runs multiple parallel environments for efficient data collection
- **Two Variants**: PPO-Penalty (adaptive KL penalty) and PPO-Clip (clipped objective, more commonly used)

## Algorithm (PPO-Clip)
```
L^CLIP(θ) = E_t[min(r_t(θ) * A_t, clip(r_t(θ), 1-ε, 1+ε) * A_t)]
where r_t(θ) = π_θ(a_t|s_t) / π_θ_old(a_t|s_t)
```

## Results
- Matches or exceeds performance of TRPO (Trust Region Policy Optimization) on continuous control tasks
- More stable training than vanilla policy gradient methods
- Became the go-to algorithm for OpenAI's robotics and game-playing research
- Successfully applied to Dota 2 (OpenAI Five) and many other domains

## Relevance to Orbit Wars
- **Primary RL algorithm** recommended for Orbit Wars — PPO is the most proven method for game AI
- **Stable-Baselines3** provides a production-ready PPO implementation
- **Clipped objective** prevents catastrophic policy updates that could destroy a working agent
- **Sample efficiency** is important given limited Kaggle GPU time
- **Multiple epochs** allow extracting more learning from each batch of game episodes
- **Recommended hyperparameters for Orbit Wars**:
  - Learning rate: 3e-4
  - Clip range (ε): 0.2
  - GAE λ: 0.95
  - Entropy coefficient: 0.01
  - Batch size: 2048-8192
  - Epochs per update: 3-10
