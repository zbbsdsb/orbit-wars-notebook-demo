# Dream to Control: Learning Behaviors by Latent Imagination

**Authors**: Danijar Hafner, Timothy Lillicrap, Ian Fischer, Ruben Villegas, David Ha, Honglak Lee, James Davidson
**Institution**: Google Brain, University of Toronto
**Year**: 2020
**arXiv**: [1912.01603](https://arxiv.org/abs/1912.01603)

## Core Contribution
A reinforcement learning algorithm (Dreamer) that learns a world model from experience and trains a policy entirely within the learned model's imagination. This dramatically improves sample efficiency by learning behaviors without interacting with the real environment.

## Key Techniques
- **World Model**: Learns a compact latent representation of the environment using a recurrent state-space model (RSSM)
  - **Encoder**: Maps observations to latent states
  - **Transition Model**: Predicts next latent state given current state and action
  - **Reward Model**: Predicts reward from latent state
  - **Decoder**: Reconstructs observations from latent state (for visualization)
- **RSSM (Recurrent State-Space Model)**: Combines deterministic and stochastic latent states for robust long-term predictions
  - Deterministic component: h_t = f(h_{t-1}, z_{t-1}, a_{t-1})
  - Stochastic component: z_t ~ p(z_t | h_t)
- **Actor-Critic in Imagination**: Both policy and value networks are trained entirely inside the imagined trajectories generated by the world model
- **Imagination Rollouts**: Unroll the world model for multiple steps to generate imagined trajectories for policy optimization
- **Backpropagation Through Time**: Gradients flow through the imagined trajectories to update the policy

## Algorithm
```
# 1. Collect real experience
experience = env.step(action)

# 2. Update world model
world_model.train(experience)  # encoder, RSSM, decoder, reward model

# 3. Imagine trajectories
imagined = world_model.imagine(initial_state, horizon=H)

# 4. Train actor-critic on imagined trajectories
actor_critic.train(imagined)
```

## Results
- Achieved state-of-the-art sample efficiency on 20+ continuous control tasks
- Solved tasks with 10-100x fewer environment interactions than model-free methods
- Successfully learned behaviors from images (pixel observations)
- Comparable final performance to model-free methods despite using far fewer samples

## Relevance to Orbit Wars
- **Sample efficiency** is critical — Dreamer could train strong Orbit Wars agents with far fewer games
- **World model for Orbit Wars**: Learn to predict planet positions, fleet movements, and battle outcomes
- **Imagination training**: Generate thousands of imagined game scenarios without running actual games
- **RSSM for temporal modeling**: Capture the temporal dynamics of Orbit Wars (planet rotation, fleet movement)
- **Challenge**: Orbit Wars has discrete actions and complex multi-agent dynamics, which may be harder to model than continuous control
- **Potential approach**: Use Dreamer for rapid prototyping of strategies, then fine-tune with PPO on real games
- **Compute trade-off**: World model training is expensive but amortized — once trained, policy optimization is very cheap
