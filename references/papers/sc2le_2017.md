# StarCraft II: A New Challenge for Reinforcement Learning

**Authors**: Oriol Vinyals, Timo Ewalds, Sergey Bartunov, Petko Georgiev, Alexander Vezhnevets, Michelle Yeo, Alireza Makhzani, Heinrich Küttler, John Agapiw, Julian Schrittwieser, Jon Close, Alex Budden, Erich Elsen, Sylvain Gelly, Stephan Gouws, Thomas Hubert, Filip Karczewski, James Molloy, Tom Schaul, Daniel Visentin, Georg Ostrovski, Stig Petersen, Faustino Gomez, Tim Hertel, David Silver
**Institution**: DeepMind Technologies, Blizzard Entertainment
**Year**: 2017
**arXiv**: [1708.04782](https://arxiv.org/abs/1708.04782)

## Core Contribution
Introduced SC2LE (StarCraft II Learning Environment), a large-scale reinforcement learning environment based on StarCraft II. Defined the problem of playing StarCraft II as an RL challenge and established benchmark tasks of varying difficulty.

## Key Techniques
- **Feature Layers**: Spatial observations rendered as feature layers (similar to screen pixels but with semantic meaning)
- **Action Space**: Structured action space with function IDs and arguments (point, unit, queued)
- **Mini-Game Tasks**: 14 mini-games of increasing difficulty (e.g., "Move to Beacon", "Collect Mineral Shards", "Defeat Roaches")
- **Full Game**: Complete 1v1 StarCraft II gameplay with build orders, unit control, and map exploration
- **PySC2 API**: Python interface for interacting with the StarCraft II engine
- **Replay Packs**: Large datasets of human replays for imitation learning

## Results
- Established baseline performance for RL agents on mini-games
- Demonstrated that even simple mini-games require sophisticated RL techniques
- Full-game play remained far from human level at time of publication
- Paved the way for AlphaStar research

## Relevance to Orbit Wars
- **Structured action space** design is very similar to Orbit Wars (function ID + arguments = planet selection + angle + fleet size)
- **Feature layers** concept maps to our state representation design (encoding game state as structured features)
- **Mini-game curriculum** approach directly applicable — start with simple Orbit Wars scenarios before full games
- **Replay data for imitation learning** — same approach can be used with Orbit Wars episode datasets
- **Scale of challenge** — SC2LE demonstrated that RTS games require hierarchical decision-making, relevant to Orbit Wars
- **PySC2 architecture** provides design inspiration for our Orbit Wars environment wrapper
